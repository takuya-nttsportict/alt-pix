# CLAUDE.md

このファイルは alt-pix プロジェクトにおける開発の原則（プリンシパル）を定義する。
Claude は本プロジェクトで作業する際、**必ず**この原則に従うこと。

---

## プロジェクト概要

バレーボール映像（SRT/RTMP ライブ配信および MP4 バッチ）から、ボール位置と
選手（背番号単位）をトラッキングするシステム。最終目標は映像フレーミング
（自動ズーム・カメラ追従）への活用。GPU 前提。

---

## 開発プリンシパル

### 1. フェーズ駆動開発

- 開発は**フェーズ単位**で進める。
- **各フェーズごとにプルリクエスト（PR）を作成し、レビュー・マージしてから次へ進む。**
- フェーズの作業は専用ブランチで行い、`main` へは PR 経由でのみマージする。
- **新しいフェーズのブランチは、必ず最新の `main`（前フェーズがマージ済み）から分岐する。**
  - 前フェーズが未マージのまま `main` から分岐すると成果物が欠落する。これを厳禁とする。

### 2. フェーズごとの評価とフィードバック

- **各フェーズの完了時に必ず評価を行う。**1 フェーズ ＝ 1 評価ポイントとなるよう設計する。
- 評価は再現可能な手順（評価スクリプト・指標）とともに提示する。
- 評価結果に基づき方針をアップデートする。

### 3. 方針アップデート ＝ 後続フェーズの見直し

- 評価で得た知見は、**後続フェーズの計画見直し**として反映する。
- 方針変更は記録し、なぜ変えたのかを残す。

### 4. 技術選定

- **実装可能なすべての技術を俎上に並べ、比較したうえでリーズナブルなものを選ぶ。**
  - 「思いついたものをとりあえず試す」カットアンドトライ的な技術選定を禁止する。
  - 選定時は候補一覧・各々の長所短所・選定理由を文書化する。
- **Claude の独断で、合意済みの技術選定を勝手に変更しない。**
  - 変更が必要と考える場合は、根拠を示してユーザーに諮る。

### 5. ライセンス・商用利用

- 選定する技術は**商用利用可能**なものとする。
- ロイヤリティが必要なものは都度判断するが、**よほどのことがない限りオープンソースの代替技術を使う。**
  - 例：Ultralytics YOLO（AGPL-3.0）は採用しない → YOLOX（Apache 2.0）等を使う。
- 各依存ライブラリのライセンスを把握し、記録する。

### 6. 依存関係の事前設計

- **Python の依存関係問題をカットアンドトライで解決しない。**
- 採用前に、各パッケージの依存（特に numpy ABI、CUDA バージョン、torch との整合）を
  調査し、競合しない組み合わせを設計してから実装する。
- 環境は Dockerfile で固め、再現性を担保する。

### 7. デバッガビリティ

- **すべてのシステムで、ログと処理パフォーマンス表示を考慮した実装にする。**
  - 標準のログレベル（DEBUG/INFO/WARNING/ERROR）を実装する。
  - DEBUG: フレーム単位の生値。INFO: 区間サマリ（fps、検出率等）。
    WARNING: 回復可能な異常。ERROR: 致命的エラー。
  - 処理時間・fps・各段の検出数など、性能と挙動を観測できる出力を必ず用意する。

## ブランチ運用

| ブランチ | 用途 |
|---|---|
| `main` | 各フェーズのマージ先。常に評価済み・動作する状態を保つ。 |
| `claude/phaseN-*` | フェーズ N の作業ブランチ。PR 経由で `main` にマージ。 |

- 作業開始時に `git fetch origin && git checkout -b claude/phaseN-xxx origin/main` で
  最新 main から分岐する。

## フェーズ計画（現時点）

> 評価結果に応じて後続フェーズは見直す（プリンシパル 2・3）。

- **Phase 1**: 基盤（ストリーム受信・人物検出・初期トラッキング） … マージ済み
- **Phase 2**: ボール検出・コートキャリブレーション・ロギング整備 … 評価・マージ調整中
- **Phase 3**: ボール検出精度の改善（最優先）
- **Phase 4**: 背番号 OCR
- **Phase 5**: 選手トラッキング高度化（Re-ID 等。調査結果はドキュメントに保持）
- **Phase 6**: 統合・最適化・フレーミング活用

| 実行環境 | Docker (nvcr.io/nvidia/pytorch) | - |
