# Phase 4（再定義）: コート上選手の知覚 — 技術選定

> プリンシパル 3・4 に基づく記録。Phase 3 評価とユーザーとの議論を経て、
> 旧 Phase 4（背番号 OCR）・旧 Phase 5（選手トラッキング高度化）を見直し、
> **フレーミングと試合分析の共通知覚コア**として再定義した。

## 背景と問題の再定義

### きっかけ

- 当初計画では Phase 4 = 背番号 OCR、Phase 5 = ReID による選手トラッキング高度化。
- しかし最終目標（フレーミング）から逆算すると **個人識別は不要**であり、
  さらに「フィールド選手／控え／審判の区別」は OCR でも ReID でも解けないことが
  議論で明確になった。
- 加えて、本フレームワークは将来 **試合分析（アナリティクス）** にも使う方針。
  フレーミングとは処理量・レイテンシ・出力が別だが、**知覚コアは共通**。

### 二つのプロダクトと共通コア

```
        ┌─────────────────────────────────────┐
        │   共通知覚コア (Perception Core)      │
        │  ・ボール検出 (TrackNet / Phase 3)    │
        │  ・人物検出・トラッキング (MOT)        │
        │  ・コートキャリブレーション (Phase 2) │
        │  ・チーム分類・役割フィルタ           │
        └──────────────┬──────────────────────┘
                       │ 構造化出力 (JSONL: 位置・track_id・team・role)
          ┌────────────┴────────────┐
          ▼                         ▼
  ┌───────────────┐         ┌──────────────────┐
  │ フレーミング   │         │ 試合分析          │
  │ 低遅延・追従    │         │ バッチ・高精度    │
  │ 個人識別 不要   │         │ 個人識別 必要 ★   │
  └───────────────┘         └──────────────────┘
```

**背番号 OCR の位置づけ**: フレーミングには不要だが、試合分析では個人識別に必須。
よって **キャンセルではなく、試合分析フェーズ（Phase 6）まで保留**とする。

## 技術選定（プリンシパル 4：候補比較）

ライセンスは各 GitHub の LICENSE ファイルで検証済み。**コードのライセンスと
学習済み重み・データセットのライセンスは別物**である点に注意（許諾的コードでも
MOT17/Duke/SoccerNet/SVHN 学習済み重みは商用不可）。

### 1. 多物体トラッキング（MOT）

| 候補 | ライセンス | ReID 要否 | 判定 |
|---|---|---|---|
| **roboflow/trackers（採用済）** | Apache 2.0 | 内蔵（ByteTrack/OC-SORT/BoT-SORT） | **採用継続** |
| Norfair | BSD-3-Clause | 不要 | 軽量代替として保持 |
| ByteTrack / OC-SORT / BoT-SORT | MIT | 動きベースは不要 | roboflow/trackers 経由で利用 |
| StrongSORT | GPL-3.0 | 要 | ✗ コピーレフト |
| BoxMOT | AGPL-3.0 | — | ✗ 除外 |

→ **既存の roboflow/trackers（Apache 2.0）で十分**。新規導入不要。

### 2. チーム分類（2 チームへのクラスタリング）

| 候補 | ライセンス | 判定 |
|---|---|---|
| **SigLIP → KMeans（roboflow/sports 流）** | コード MIT＋SigLIP Apache 2.0 | **採用**。ユニフォーム埋め込みで 2 クラスタ化 |
| HSV ヒストグラム＋KMeans | 自前（ライセンス露出なし） | 軽量フォールバック |
| CLIP / OpenCLIP ＋ KMeans | 重みの配布条項が曖昧 | △ SigLIP を優先 |

### 3. 役割区別（フィールド／控え／審判）

| 手法 | 実装 | 判定 |
|---|---|---|
| **(c) コート幾何ヒューリスティック** | 自前（Phase 2 キャリブ資産） | **採用（主軸）**。場内/場外で控え・審判を分離。画角変化に強い＝普遍的 |
| **(b) チーム色クラスタの外れ値＝審判** | SigLIP クラスタリング上 | **採用（補助）** |
| (a) 検出器に審判クラスを学習 | 自前アノテーション | 将来オプション（商用オープンデータに「ベンチ」クラスは存在しない） |

> 調査結論：役割区別の turnkey 商用モデルは存在せず、**(b)＋(c) の組み合わせが定石**。
> SoccerNet-GSR は役割ラベルを持つが研究用（NDA）で商用不可。

### 4. 姿勢推定（任意）

| 候補 | ライセンス | 用途 |
|---|---|---|
| mmpose / RTMPose | Apache 2.0 | サーブ局面の文脈推定、OCR の番号位置特定（後続） |

### 5. 背番号 OCR（**Phase 6 まで保留**）

| 候補 | ライセンス | 備考 |
|---|---|---|
| EasyOCR（現選定） | Apache 2.0 | 検証済み商用可。生のジャージ番号には弱く要 fine-tune |
| PARSeq | Apache 2.0 | 短文字列に強い STR バックボーン |
| jersey-number-pipeline | **無ライセンス（all rights reserved）** | アーキテクチャ参考のみ。再実装すること |

> SOTA は単フレーム OCR ではなく **tracklet 単位の時間方向多数決**
> （legibility 判定 → STR → トラック全体で投票）。トラッキングと統合して初めて機能する。
> SoccerNet/SVHN 学習済み重みは商用不可 → 自前データで学習。

### 除外確定（ライセンス）

Ultralytics YOLO（AGPL）、BoxMOT（AGPL）、StrongSORT（GPL）、SportsLabKit（GPL）、
volleyball_analytics（GPL）、VolleyVision（無ライセンス＋AGPL 依存）。

## 普遍性の担保（入力条件変化への耐性）

ユーザー懸念（画角・サイズが変わり得る）への対応：

- **役割区別はコート幾何で解く** → キャリブし直すだけで成立。映像依存の学習を避ける。
- **チーム分類は埋め込み＋クラスタリング** → 色・画角に対して相対的、特定映像に過学習しない。
- **検出器・OCR の重みは自前データ／汎用バックボーン**で学習 → 配布条項と汎化の両立。
